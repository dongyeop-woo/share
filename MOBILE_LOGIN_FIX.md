# 모바일 로그인 "load failed" 문제 해결

## 문제 분석

모바일에서만 발생하는 로그인 실패는 일반적으로:
1. **쿠키/세션 문제** - 모바일 브라우저의 쿠키 정책이 더 엄격함
2. **CORS 문제** - 모바일 브라우저가 다른 방식으로 CORS 처리
3. **네트워크 타임아웃** - 모바일 네트워크가 느림

## 현재 설정

- `same-site: none` + `secure: true` - 모바일에서 문제 발생 가능
- `credentials: "include"` - 프론트엔드에서 쿠키 포함 요청

## 해결 방법

### 1. 쿠키 설정 최적화 (application.yml)

현재 설정은 이미 최적화되어 있습니다:
- `secure: true` - HTTPS 필수
- `same-site: none` - 크로스 사이트 요청 허용
- `http-only: true` - XSS 방지

### 2. 프론트엔드 에러 처리 확인

모바일에서 발생하는 정확한 에러 확인 필요:
- 브라우저 개발자 도구 확인
- 네트워크 탭에서 실패한 요청 확인

### 3. 세션 타임아웃 증가

이미 추가했습니다 (86400초 = 24시간)

## 다음 단계

1. 브라우저 개발자 도구에서 정확한 에러 확인
2. 네트워크 탭에서 실패한 요청 확인
3. 쿠키가 설정되는지 확인
